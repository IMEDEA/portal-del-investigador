Instalación

Estas instrucciones indican cómo configurar una máquina de desarrollo con Ubuntu.

  • Clonar el repositorio
git clone --recursive https://github.com/tic-ull/investigacion.git
  • Instalar dependencias
Para la instalación de dependencias de módulos python se usa pip. Se recomienda usar esta herramienta junto con virtualenv, aunque en las instrucciones no se entrará a explicar su funcionamiento. 
sudo apt-get install python-pip
sudo pip install -U distribute
sudo apt-get install libmysqlclient-dev postgresql postgresql-client postgresql-contrib \
                     libpq-dev libxml2-dev libxslt1-dev python-dev xvfb
sudo pip install -r requirements.txt
  • Configurar aplicación

Tanto el proyecto como las aplicaciones individuales se configuran a través de su fichero settings.py. La configuración de estos ficheros puede ser sobreescrita a través de los ficheros settings_local.py. En las aplicaciones con valores de configuración que necesitan ser sobreescritos obligatoriamente se encuentra un fichero de ejemplo (settings_local-sample.py) que define los valores a configurar, y que se puede usar como punto de partida: 

cp settings_local-sample.py settings_local.py

Configuración de Investigación (investigacion/settings_local.py)

VARIABLE DESCRIPCIÓN
BASE_URL URL de la página principal del proyecto. Por defecto es un servicio externo.
SECRET_KEY https://docs.djangoproject.com/en/1.7/ref/settings/#secret-key
DEBUG https://docs.djangoproject.com/en/1.7/ref/settings/#std:setting-DEBUG
TEMPLATE_DEBUG https://docs.djangoproject.com/en/1.7/ref/settings/#template-debug
DEVEL Si DEVEL = True, el servidor de aplicaciones de Django servirá los ficheros estáticos de los que hace uso el proyecto. En otro caso deberá hacerse desde un servidor externo.
ADMINS https://docs.djangoproject.com/en/1.7/ref/settings/#std:setting-ADMINS
SUPPORT Nombre del Departamento de Soporte (mostrado al usuario en caso de error)
EMAIL_SUPPORT Correo Electrónico del Departamento de Soporte (mostrado al usuario en caso de error)
CAS_SERVER_URL URL del Servicio de Autenticación (CAS) //link_a_pag_cas//
WS_SERVER_URL URL base del webservice //link_a_pag_ws//

Configuración de CVN (cvn/settings_local.py)

VARIABLE DESCRIPCIÓN
UNIVERSITY Nombre de la universidad
EXPIRY_DATE Tipo datetime.date. Todos los CVNs con fecha de generación anterior a su valor se consideran caducados.
FECYT_USER Nombre de usuario para acceder a los webservices de la FECYT
FECYT_PASSWORD Contraseña para acceder a los webservices de la FECYT
CVN_PDF_AUTHOR_NOAUT Al generar un CVN-PDF a través del webservice de la FECYT se escribe en sus metadatos, en el campo author, el identificador de la universidad que lo genera. A través de esta variable se puede impedir subir PDFs generados por una universidad concreta. Útil para obligar al usuario a pasar por el editor de la FECYT para que complete su CVN.
HISTORICAL_YEAR Se usa para seleccionar qué base de datos histórica se usará para mostrar el informe de producción científica (CVN-PDF del usuario GesInv-ULL). Los valores permitidos son las claves del diccionario HISTORICAL. Más información en Base de datos histórica.
  • Configurar bases de datos
El sistema de base de datos recomendado es PostgreSQL. No se entrará a detallar la configuración del sistema de base de datos y la creación de usuario/base de datos, ya que es el método estándar usado en cualquier proyecto Django.

Base de datos por defecto

La base de datos puede configurarse sobreescribiendo los valores por defecto en investigacion/settings_local.py siguiendo el formato que indica la documentación de Django. Después de configurar los valores se debe ejecutar: 

python manage.py makemigrations
python manage.py migrate

Base de datos histórica

Para la realización de Informes de Producción Científica extraídos de los CVN se puede congelar la base de datos por defecto a una base de datos histórica de sólo lectura. Cada base de datos histórica se debe añadir a la configuración de base de datos de Django en investigacion/settings_local.py, y posteriormente al diccionario HISTORICAL con el siguiente formato: 

HISTORICAL = {'2001': 'nombre_bbdd_historica_2001',
              '2002': 'nombre_bbdd_historica_2002'
              ...
}
